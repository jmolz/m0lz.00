# Feature: TOC Simplification + Mobile Overflow Fix

> Generated by `/plan-feature` — read CLAUDE.md and referenced files before implementing.

## Overview

Simplify the Table of Contents in both the floating TOC and research panel to show only top-level sections (h2), fix the panel TOC so it scrolls independently of the content, and eliminate horizontal scrolling on mobile research pages.

## User Story

As a reader browsing research content, I want a scannable table of contents that I can scroll independently from the article, and on mobile I want content that stays within the viewport without horizontal scrolling, so that I can navigate long research documents intuitively on any device.

## Success Criteria

- [ ] TOC shows only h2-level headings (sections), not h3 subsections
- [ ] Panel TOC scrolls independently — does not require scrolling past content sections
- [ ] Research pages have no horizontal scroll on mobile (393px viewport)
- [ ] All validation commands pass with zero errors
- [ ] No regressions in existing functionality

### Task 3: Fix panel TOC independent scrolling

**File:** `components/research-panel.tsx`
**Action:** MODIFY
**Description:** The panel TOC sidebar uses `max-h-full` (line 176) which sizes relative to the flex parent's content height — making the TOC as tall as the article with nothing to scroll independently. Replace with a viewport-relative max-height so the TOC is constrained to the visible area and scrolls within it.

**Change:**
- Line 176: Replace `max-h-full` with `max-h-[calc(100vh-4rem)]` on the `<nav>` element

The `4rem` accounts for the panel's top padding (`py-8` = 2rem top) plus breathing room. Combined with `sticky top-0 self-start`, the TOC will stick to the top of the panel viewport and scroll independently when its content exceeds the viewport height.

**Depends on:** Task 2 (fewer items means less scrolling needed, but the fix is independent)
**Validate:** `npm run dev` → open a writing post → click "RESEARCH ↗" → verify the panel TOC scrolls independently on the right side

### Task 4: Fix mobile horizontal scroll on research pages

**File:** `app/globals.css`
**Action:** MODIFY
**Description:** Add `overflow-wrap: break-word` to `.prose-content` to prevent long words, URLs, or formula text from pushing the viewport wider than the screen on mobile. This is the root cause of the horizontal scrolling seen on iPhone 16 Pro (393px viewport).

**Changes:**
Add to the existing `.prose-content` rules (after line 125):

```css
.prose-content {
  overflow-wrap: break-word;
}
```

Also add a catch-all for block-level children that might overflow:

```css
.prose-content pre,
.prose-content table,
.prose-content img {
  max-width: 100%;
}
```

Note: `pre` elements already have `overflow-x: auto` from the rehype-pretty-code rules (line 61), so code blocks will scroll internally rather than push the page wider. The `max-width: 100%` is a safety net for any content without explicit overflow handling.

**Depends on:** none
**Validate:** `npm run dev` → open `/research/pice` on a mobile viewport (393px) → verify no horizontal scrolling

### Task 5: Add overflow-x containment to research page wrapper

**File:** `app/research/[slug]/page.tsx`
**Action:** MODIFY
**Description:** Add `overflow-x-hidden` to the research page's outer container div as a belt-and-suspenders fix. This ensures no child element (formulas, inline code, tables) can push the page wider even if `overflow-wrap` doesn't catch it.

**Change:**
- Line 42: Add `overflow-x-hidden` to the className:
  `"max-w-3xl mx-auto px-6 py-16"` → `"max-w-3xl mx-auto px-6 py-16 overflow-x-hidden"`

**Depends on:** none

## Notes

- The PICE research frontmatter defines 8 sections; MCP-Guard has 2. These are all h2-level. The h3 subsections were being auto-discovered from the MDX content, making the PICE TOC ~30+ items long.
- The `sticky top-0` positioning on the panel TOC already handles viewport anchoring — the only missing piece was the viewport-relative height constraint.
- The mobile horizontal scroll is likely caused by mathematical formula text (inline LaTeX-style notation) in the PICE research document. The `overflow-wrap: break-word` fix handles the text wrapping, and `overflow-x-hidden` on the container is a safety net for any remaining wide elements.
- Both the floating `TableOfContents` and the panel-scoped TOC in `ResearchPanel` have separate `TocItem` interfaces — they could be unified into a shared type in the future, but that's out of scope for this fix.
