# Feature: Repo Rename + Catalog Expansion

> Generated by `/plan-feature` ‚Äî read CLAUDE.md and referenced files before implementing.

## Overview

Rename 3 GitHub repos to their m0lz catalog IDs (`pice-framework` ‚Üí `m0lz.02`, `mcp-guard` ‚Üí `m0lz.03`, `case-pilot` ‚Üí `m0lz.04`), add a new `case-pilot` branch mark variant, promote Case Pilot into the catalog, and update all codebase references.

## User Story

As a visitor to m0lz.dev, I want the project namespace to be consistent across GitHub repos, the website, and documentation, so that the m0lz catalog identity is coherent everywhere.

- [ ] No references to `pice-framework`, `mcp-guard`, or `case-pilot` remain as repo names
- [ ] All validation commands pass with zero errors
- [ ] No regressions in existing functionality
