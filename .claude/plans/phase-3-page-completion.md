
**Section label pattern (from landing page):**

```tsx
<p className="text-xs tracking-widest text-[var(--muted)] mb-6">
  SECTION LABEL
</p>
```

**Divider pattern:**

```tsx
<hr className="border-[var(--border)] border-t-[0.5px]" />
```

**Link pattern (from footer):**

```tsx
<a
  href="https://..."
  target="_blank"
  rel="noopener noreferrer"
  className="underline underline-offset-4 decoration-[var(--border)] hover:decoration-[var(--foreground)] transition-colors"
>
  Text
</a>
```

**Color tokens — ONLY these four:**
- `var(--foreground)` — primary text
- `var(--background)` — page background
- `var(--muted)` — secondary text, labels, metadata
- `var(--border)` — dividers, borders

**Font weights — ONLY 400 and 500:**
- `font-medium` (500) for emphasis
- Default (400) for everything else
- NEVER use `font-semibold` or `font-bold`

**Import style:**

```tsx
import type { Metadata } from 'next'        // React/Next
import { BranchMark } from '@/components/...' // internal, @/ alias
import { projects } from '@/data/...'         // data
```

## Critical Design Decisions

### About Page Content Strategy

The PRD specifies these content blocks for the About page:

1. **Bio** — "AI Product Engineer & Technical Founder"
2. **Stats** — "15+ shipped products, 7,100+ GitHub contributions/year"
3. **Career** — "Previously: 15 years B2B SaaS leadership (VP/SVP/CSO) at 24/7 Software, CleanAir.ai"
4. **Education** — "MBA — Nova Southeastern University, BSBA Entrepreneurship — UCF"
5. **Links** — GitHub, LinkedIn
6. **m0lz catalog** — Mention of PICE framework and catalog system

Layout approach: Use the same sparse, austere structure as other pages. Sections separated by `<hr>` dividers. No headshots, no decorative elements, no accent colors. Let the content speak.

### Research Page Strategy

PRD says "placeholder for future deep dives." Current stub is bare "Coming soon." — improve to a purposeful placeholder that tells visitors what will go here and why it exists as a separate section from Writing.

### Metadata Strategy

Currently, pages have titles but most lack descriptions. Add `description` to every page's `metadata` export for SEO and social sharing. Descriptions should be concise (under 160 chars) and relevant.

Pages needing metadata enrichment:
- `/` — has title only, needs description ✅ (layout has it, but page metadata should be explicit)
- `/writing` — needs description
- `/projects` — needs description
- `/research` — needs description
- `/about` — needs description
- `/writing/[slug]` — already has per-post description ✅
