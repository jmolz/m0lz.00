# Feature: Phase 1 — Foundation

> Generated by `/plan-feature` — read CLAUDE.md and referenced files before implementing.

## Overview

Scaffold the m0lz.dev project from scratch: Next.js 16 with App Router, Tailwind CSS 4, Geist fonts, monochrome design system, branch mark identity, dark/light theme, and all route stubs. This is the foundation everything else builds on — no content infrastructure, no MDX, no project data.

## User Story

As a developer visiting m0lz.dev, I want to see a clean, monochrome site shell with navigation, theme toggle, and all pages reachable, so that the site is ready for content and project data to be added in subsequent phases.

**File:** `postcss.config.mjs`, `app/globals.css`
**Action:** CREATE / MODIFY
**Description:** Install `tailwindcss`, `@tailwindcss/postcss`, and `postcss`. Create `postcss.config.mjs` with the `@tailwindcss/postcss` plugin. Replace the contents of `app/globals.css` with the Tailwind import and monochrome design tokens.

**Important:** Tailwind CSS 4 does NOT use `tailwind.config.ts`. All theme customization is done in CSS via `@theme`. Delete `tailwind.config.ts` if `create-next-app` generates one.

```css
@import "tailwindcss";

@theme {
  --font-sans: 'Geist Sans', ui-sans-serif, system-ui, sans-serif;
  --font-mono: 'Geist Mono', ui-monospace, monospace;
}

:root {
  --foreground: #000000;
  --background: #ffffff;
  --muted: #737373;
  --border: #e5e5e5;
}

.dark {
  --foreground: #ffffff;
  --background: #000000;
  --muted: #a3a3a3;
  --border: #262626;
}

body {
  color: var(--foreground);
  background-color: var(--background);
  font-family: var(--font-sans);
}
```

**Validate:** `npm run dev` loads with correct fonts and colors

### Task 4: Create Theme Provider and Toggle

**File:** `components/theme-provider.tsx`, `components/theme-toggle.tsx`
**Action:** CREATE
**Description:**

**theme-provider.tsx**: A client component that:
1. Reads theme preference from `localStorage` on mount (default: `'dark'`)
2. Applies `dark` class to `<html>` element
3. Provides theme context (`theme`, `toggleTheme`) to children
4. Includes a blocking `<script>` injected via `dangerouslySetInnerHTML` in the root layout's `<head>` to prevent flash of wrong theme. The script reads `localStorage` synchronously and applies the `dark` class before first paint.

**theme-toggle.tsx**: A client component (`'use client'`) that:
1. Renders a minimal button (moon/sun icon or simple text toggle)
2. Calls `toggleTheme()` from context
3. Monochrome styling only — uses `--foreground` and `--muted` colors

**Depends on:** Task 2
**Validate:** Toggle switches theme instantly; refresh preserves preference; no flash of wrong theme on load

### Task 5: Build Branch Mark SVG Component

**File:** `components/branch-mark.tsx`
**Action:** CREATE
**Description:** Create an SVG component that renders the branch mark identity system. The component is a server component (no `'use client'`).

Props:
- `variant`: `'personal' | 'blog-agent' | 'pice' | 'mcp-guard'`
- `size`: number (default 36)
- `className`: optional string

Structure:
- Rounded rectangle (`rx` for rounded corners)
- Two vertical lines (trunks) — always present, centered
- Horizontal lines (branches) — positioned per variant spec:
  - **personal**: 1 left branch on left trunk (upper), 1 right branch on right trunk (lower)
  - **blog-agent**: 1 left branch on left trunk, 2 right branches on right trunk
  - **pice**: 2 left branches on left trunk, 1 right branch on right trunk
  - **mcp-guard**: 1 left + 1 right branch, mirrored at same height

Colors:
- Rounded rect uses `currentColor` (fill)
- Trunks and branches use the inverse color (stroke or fill)
- In dark mode (white background rect, black lines) and light mode (black rect, white lines), this is handled by using `currentColor` for the rect fill and a contrasting color for the inner elements. The simplest approach: rect fill = `currentColor`, inner lines = use CSS `color` on the parent to control the scheme.

Stroke weight: 2px at 36px size, scales proportionally (`strokeWidth = (2 / 36) * size`).

Must be legible at 16px (favicon) through 64px+ (social cards).

**Depends on:** none (pure SVG, no style dependencies)
**Validate:** Render all 4 variants at sizes 16, 36, 64 — visually confirm distinct patterns and legibility

## Notes

- **Tailwind CSS 4 breaking change**: No `tailwind.config.ts`. All theme customization is done via CSS `@theme` directive in `globals.css`. The PRD references `tailwind.config.ts` but this file should NOT be created.
- **`create-next-app` in Next.js 16**: May generate boilerplate that needs cleanup (default page content, default styles, default favicon). Strip everything back to the bare shell.
- **Theme flash prevention**: The blocking script approach is critical for dark-mode-default sites. It must run synchronously before first paint to apply the `dark` class. This is a common pattern — Next.js's `suppressHydrationWarning` on `<html>` prevents hydration mismatch warnings.
- **Branch mark SVG**: Using `currentColor` approach means the component adapts to theme without needing theme context. The parent element's `color` CSS property controls the fill.
- **Deferred to Phase 2**: MDX pipeline, post components, project data, code blocks. This phase is strictly about the shell.
- **Deferred to Phase 4**: Vercel domain connection (m0lz.dev), OG images, RSS, SEO metadata.
