# Feature: Phase 4 — Polish & Deploy

> Generated by `/plan-feature` — read CLAUDE.md and referenced files before implementing.

## Overview

Ship production-ready polish: Open Graph image generation for social sharing, RSS feed for subscribers, enhanced metadata, and performance validation. This is the final implementation phase before launch.

## User Story

As a visitor or subscriber, I want OG images that render on social platforms, an RSS feed I can subscribe to, and fast page loads, so that the site looks professional when shared and is easy to follow.

## Implementation Tasks

Execute in order, top to bottom. Each task is atomic.

### Task 1: Build OG Image Infrastructure

**File:** `lib/og.tsx`
**Action:** CREATE
**Description:** Create shared OG image utilities:

1. **Font loader** — Read Geist Sans Regular (400) and Medium (500) `.ttf` files from `node_modules/geist/dist/fonts/geist-sans/`. Use `fs.readFileSync` at module level. Export as `geistRegular` and `geistMedium` ArrayBuffers for Satori.

2. **Branch mark SVG helper** — Export a function `getBranchMarkDataUri()` that returns a base64 data URI of the personal branch mark SVG. Hardcode the dark-theme colors (white fill on black background) since OG images are always dark. Use the exact geometry from `public/favicon.svg` but at 64x64 size:
   - Rounded rect background (fill white)
   - Two vertical trunk lines (stroke black)
   - Two horizontal branch lines at y=0.35 and y=0.65 (stroke black)
   - Wait — the OG image has dark background. So invert: rounded rect fill `#ffffff`, strokes `#000000`. Actually the OG bg is black, so the branch mark should have a white rounded rect with black strokes to contrast. Check the favicon geometry and scale up proportionally.

3. **Shared OG layout** — Export a function `createOGImage(props: { title: string; subtitle?: string })` that returns an `ImageResponse`. The layout:
   - 1200x630 dimensions
   - Black (`#000000`) background
   - White (`#ffffff`) text
   - Muted text: `#a3a3a3` (dark theme --muted value)
   - Branch mark at top-left (64x64) rendered via `<img>` with the data URI
   - Title below the mark, Geist Medium, ~40px
   - Subtitle below title (if provided), Geist Regular, ~20px, muted color
   - Bottom-right: "m0lz.dev" in small muted text

**Depends on:** none
**Validate:** `npx tsc --noEmit` (types check)

### Task 5: Build RSS Feed

**File:** `app/feed.xml/route.ts`
**Action:** CREATE
**Description:** Static route handler that generates an RSS 2.0 feed at build time. With `output: 'export'`, this produces a static `feed.xml` file in the build output.

Requirements:
- Export only a `GET` function (no other HTTP methods)
- Do NOT read from the `Request` object (static export constraint)
- Call `getAllPosts()` to get all published posts
- Generate valid RSS 2.0 XML with:
  - `<channel>`: title "m0lz", link "https://m0lz.dev", description
  - `<item>` for each post: title, link (`https://m0lz.dev/writing/{slug}`), description, pubDate (RFC 2822 format), guid
- Return `Response` with `Content-Type: application/xml`
- XML-escape special characters in titles and descriptions (`&`, `<`, `>`, `"`, `'`)

The footer already links to `/feed.xml` — this task makes that link functional.

**Depends on:** none
**Validate:** `npm run build` — verify `feed.xml` appears in build output

## Notes

- **Static export constraint**: `output: 'export'` means route handlers cannot read from `Request`. The RSS handler must not accept or use request parameters. OG images use the `opengraph-image.tsx` file convention which generates at build time.

- **Satori font loading**: Satori (used by `next/og`) requires explicit `.ttf` font data. Load from `node_modules/geist/dist/fonts/geist-sans/Geist-Regular.ttf` and `Geist-Medium.ttf` using `fs.readFileSync`. Satori does NOT support CSS custom properties — hardcode dark theme colors in OG image JSX.

- **Branch mark in OG images**: The `BranchMark` React component uses `currentColor` and `var(--background)` which Satori doesn't support. For OG images, generate a static SVG string of the personal variant with hardcoded colors, encode as `data:image/svg+xml;base64,...`, and render via `<img>` tag in the ImageResponse JSX.

- **`metadataBase` requirement**: Next.js needs `metadataBase` set to resolve relative OG image paths to absolute URLs. Set to `https://m0lz.dev`. During development, Next.js falls back to `http://localhost:3000` automatically.

- **Deployment (out of scope)**: The PRD includes "Final Vercel deploy with m0lz.dev domain" but deployment is a manual action, not a code task. After this phase completes, run `git push origin main` to trigger Vercel auto-deploy (if connected) or deploy manually.

- **Lighthouse audit**: Target 100/100/100/100. The static site with no client-side JS beyond React hydration should score near-perfect. If Performance score is low, check for unoptimized images or large bundles.
